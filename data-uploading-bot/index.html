Background

An insurance company is outsourcing its claims investigation to a loss adjusting firm. The firm is responsible for investigating claims sent by the insurer, and settling them at a fair amount as per insurance policy.

To manage the number of claims, both companies use their own proprietary web app based systems. However, there does not exist any API for the different systems to communicate. Hence, there is duplication of work as the loss adjuster’s data entry staff needs to enter claim data in both systems.

There are also additional fields that the insurer requires that are not present in the loss adjusters system like claim imagery. These images are manually added, one by one to the insurers web-app. Depending on the case, this can take over 30 minutes.

Throughout the loss-adjusters multiple branches throughout the country, there is about a total staff of seven that is responsible for this manual work.

Objectives and Constraints

The loss adjustment firm needed to move away from manual data entry due to its relatively high cost and low accuracy. They needed an application that could automatically upload all claims data to the insurers system, including additional data that they did not hold in their system.

To achieve this, the application should be able to take additional user input. It would also need to be easily configurable so employees can use it in different branches with web app running on inaccessible on-premises servers.

Use cases for the application are also very diverse as the time, duration and complexity of the case uploading varies on a case to case basis, so the application should account for all case types.

The application should also have over 99.999% up-time during working hours and must be easy to maintain and debug in case of issue.

Solution Planning

As repeated data entry on the insurers web app was web form based and did not include any bot checks, the data entry process could easily be automated using a automation webdriver like Selenium.

The loss adjusting firms web app implementation was on on-premises servers that slightly varied from branch to branch. Hence automation webdrivers could also be used to extract claim data from the web app, independent of the web app implementation.

As the application required maximum up-time and interacted with data local to the employees computer, it was best to make the application a desktop application. Also as the application relied on employee input and bot scope definition, The application needed to have a GUI where the employee can enter additional data, check entered data as well as configure the bot.

Technical Details

User input is gathered in Tkinter based GUI application. This application includes checks for the data entered, and the ability to review entered data, including images. Entered data is combined into a user input object. The GUI application also allows defining scope of bot operation, which could include uploading data, images or parts for a given case.

Based on the claim ID number, a selenium based bot opens the requested claim forms in the firms web app running on a chrome browser. Information is then extracted from the relevant forms, specified in the GUI application. All claims data is then aggregated into a claims data object. This is then sent to another selenium based bot that enters the claims data on the insurer’s web app, based on the pre-defined scope of operation.

The application is structured on the MVC framework, that allows for easy adjustments and incremental addition of application features. It also includes detailed logging of all uploading attempts which helps in tracing bugs and other downtime issues.

The application is based in python and converted to an executable file and the application directory is packaged together in an installer executable. This can be automatically downloaded by clicking here.
